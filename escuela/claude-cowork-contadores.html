Los 3 “frentes” de Claude (sin confundirse)

Anthropic vende a Claude con varios envases. Saber cuál es cuál te ahorra plata y tiempo. Este curso cubre los tres —Cowork a fondo (módulos 2-6), Skills (7-10) y Claude Code como puerta de entrada (11)—:

Claude.ai (web)

El chat en el navegador. Lo abres en claude.ai. Bueno para consultas rápidas y subir un PDF suelto. No ve tus carpetas locales.

Claude Cowork (escritorio)

App que instalas. Ve tus carpetas, recuerda al cliente entre sesiones, soporta Projects, memoria y Skills. Es el corazón de este curso.

Claude Code (terminal)

El agente para trabajo técnico: corre en la terminal, en app de escritorio o en el navegador. Ejecuta comandos, transforma archivos en lote y construye herramientas. Lo vemos en el Módulo 11.

Si vienes del curso de IA para Contadores

Ya viste cómo pedirle cosas a Claude. Este curso es dónde: en vez de chatear desde cero cada vez, montas una sola estructura por cliente y trabajas ahí. La curva real de ahorro empieza el segundo mes.

Claude no es un solo cerebro: es una familia de modelos y tú eliges cuál usar en el selector del chat (web, Cowork y Code). La regla práctica: más capacidad = más lento y consume más de tu cuota del plan.

Por qué esto importa más a un contador que a otros profesionales

Un diseñador o un copywriter trabaja en proyectos puntuales. Un contador trabaja con los mismos 15 clientes todos los meses. Cada cliente tiene PUC propio, sector propio, histórico de ajustes, costumbres de contabilización. Si cada vez que abres Claude tienes que re-explicar “mi cliente es una PYME importadora del sector textil con responsabilidades 05-09-42-O-13”, estás perdiendo tokens y tiempo.

Cowork resuelve eso con dos cosas: carpetas locales (Claude lee tus Excel sin que los subas a cada chat) y Projects (un contenedor con instrucciones permanentes y archivos de referencia). Y cuando la misma lógica se repite en TODOS tus clientes —clasificar F1001, cuadrar IVA— la conviertes en una Skill que escribes una vez y disparas siempre. Ese es el arco del curso.

Lo clave de este módulo
  • 3 frentes: web (consultas sueltas), Cowork (tu día a día con carpetas), Claude Code (terminal y automatización).
  • 4 modelos vigentes: Fable 5 (tope), Opus 4.8 (trabajo largo), Sonnet 4.6 (día a día), Haiku 4.5 (rápido). Desconfía de nombres no oficiales.
  • El arco del curso: carpeta-por-cliente → Projects → flujos → Skills → Claude Code.

Instalar Claude Cowork y dejarlo viendo tus carpetas

Descarga, login con la cuenta de Claude Pro que ya usas, y la decisión importante: qué carpeta dejarle ver. Si le abres todo el disco, estás violando privacidad de clientes. Si no le abres nada, no sirve.

Al finalizar este módulo
  • Tener Claude Cowork instalado en Windows o Mac
  • Saber cuál carpeta-raíz darle (y por qué NO darle el escritorio completo)
  • Haber probado tu primera lectura local de un Excel sin subirlo

Paso 1: Descargar Cowork

Cowork viene en la misma cuenta de Claude. Entras a claude.ai/download con tu cuenta Pro o Max y descargas la versión para tu sistema operativo. La instalación es de doble-clic; no te pregunta nada raro.

macOS 12+Claude.dmgMismo correo de claude.ai
LinuxSolo CLI (Claude Code), no Cowork por ahora—

Si no tienes Pro

Cowork funciona con cuenta Free pero la utilidad real arranca en Pro (USD ~$20/mes). El curso Exógena con Claude Módulo 3 te ayuda a decidir el plan según cuántos clientes manejas.

Paso 2: La decisión importante — qué carpeta darle

En la primera ejecución Cowork te pregunta: ¿a qué carpeta quieres darle acceso? NO le des el escritorio completo ni Documentos enteros. Crea una carpeta dedicada para Claude.

C:\Users\TuNombre\Claude-Trabajo\   clientes\     pyme-abc\     distribuidora-xyz\     cs-quirurgica\   normativa-co\     ET-resumen.md     Res-227-exogena-2025.pdf   plantillas\     papel-trabajo-iva.xlsx

Error común

Darle C:\Users\TuNombre\ entera. Claude termina viendo fotos personales, OneDrive privado y carpetas de descargas con basura. Créale su zona y mantén ahí solo lo que es trabajo contable.

Paso 3: Tu primera lectura local

Con Cowork abierto, escribe en el chat:

Mira el archivo C:\Users\TuNombre\Claude-Trabajo\clientes\pyme-abc\balance-2025-04.xlsx y dime cuántas cuentas tiene a nivel auxiliar y cuál es el total del Activo.

Claude te va a pedir permiso para abrir el archivo (la primera vez por archivo). Le das aceptar y procesa sin que subas nada: lo lee del disco directo. Esa es la diferencia con claude.ai.

Lo clave de este módulo
  • Descarga Cowork desde claude.ai/download.
  • Crea una carpeta dedicada (NO le des Documentos completo).
  • Claude lee tus Excel del disco sin que los subas. Permiso por archivo la primera vez.

El modelo carpeta-por-cliente

Una carpeta = un cliente. Misma estructura para todos. Cuando le digas a Claude “trabaja sobre pyme-abc” sabrá adónde mirar sin que se lo deletrees. Esta es la base de todo lo que sigue.

Al finalizar este módulo
  • Tener la plantilla estándar de carpeta-cliente lista para copiar
  • Entender qué archivos viven en cada subcarpeta y por qué
  • Saber qué no meter (datos sensibles, cosas que no son contabilidad)

La plantilla estándar

Cada cliente sigue exactamente la misma estructura. Esto es lo que pega:

clientes\pyme-abc\   _cliente.md # Ficha: NIT, responsabilidades, sector, RUT, contactos   balances\     2025-04.xlsx     2025-03.xlsx   declaraciones\     iva-300\     retencion-350\     renta-110\   facturacion-electronica\     2025-FE-emitidas.csv     2025-FE-recibidas.csv   exogena\     F1001.csv     F1005.csv   requerimientos-dian\     2025-05-req-001.pdf   papeles-trabajo\

El archivo _cliente.md — la ficha que todo lo pega

Es un Markdown corto (~30 líneas) que vive en la raíz de cada cliente. Claude lo lee primero y entiende inmediatamente con quién trabaja. Plantilla:

# Cliente: PYME ABC SAS NIT: 900.123.456-7 Régimen: Ordinario Responsabilidades RUT: 05 - 07 - 09 - 42 - O-13 - 48 Sector CIIU: 4641 (Comercio textil) Tamaño NIIF: Grupo 2 (PYME) Software contable: World Office Banco principal: Bancolombia 1234-5678 Período fiscal: Enero-Diciembre Cierre mensual: Día 15 del mes siguiente ## Decisiones contables del cliente - Inventario: PEPS perpétuo - Activos fijos: línea recta, vidas útiles fiscales - Cuentas por cobrar: provisión general 1% Art. 145 ET ## Cosas que repiten cada mes - Importación de mercancía (cuenta 1465 con costos directos del Decreto 1165/2019) - Pago de comisiones a agencias internacionales (1.5% retención por servicios desde el exterior) ## Contactos - Gerente: Juana Pérez - juana@pyme-abc.com - Auxiliar contable cliente: Pedro Gómez - pedro@pyme-abc.com

Con esa ficha en frente, cualquier pregunta sobre el cliente arranca con contexto. Sin esa ficha, cada chat empieza desde cero.

Qué NO meter en la carpeta

  • Datos del titular real con cédula y dirección sin anonimizar — viola Ley 1581 si Anthropic indexa con plan equivocado. Si necesitas datos personales para una respuesta a requerimiento, úsalos en el chat de momento, no los persistas.
  • Contraseñas, tokens MUISCA, llaves AFC — nunca, ni siquiera en .txt.
  • Fotos del personal, RUT escáner del representante — no aportan al análisis y son carga de privacidad.
  • Carpetas de OneDrive o Drive sincronizadas con clientes que no son tuyos — la carpeta de Claude debe ser local pura.
Lo clave de este módulo
  • Misma estructura para todos los clientes — _cliente.md + balances + declaraciones + exogena + req-dian + papeles.
  • El archivo _cliente.md es lo que le da contexto a Claude sin que se lo expliques cada vez.
  • Nada de contraseñas, cédulas crudas ni carpetas compartidas.

Projects en Claude — contexto que no se borra

Un Project en Claude es un contenedor con Custom Instructions permanentes y archivos de referencia (Knowledge). Cada cliente tendrá su Project. Cuando entras, Claude ya sabe el PUC, las responsabilidades y cómo le gusta al cliente que se contabilicen las cosas.

Al finalizar este módulo
  • Saber crear un Project en Claude (web o Cowork)
  • Tener una plantilla de Custom Instructions para contador
  • Saber qué archivos vale la pena meter al Knowledge y cuáles no

Crear el Project — 4 clics

  1. En Cowork (o claude.ai), botón Projects en el sidebar.
  2. “New Project” → nombre: PYME ABC SAS
  3. Tab Custom Instructions → pega la plantilla de abajo.
  4. Tab Knowledge → sube los 3-5 archivos clave del cliente (PUC, último balance, RUT, última renta).

Listo. A partir de ahora, cuando le hagas chat dentro de ese Project, Claude lee tu Custom Instructions y los archivos Knowledge antes de responder. No le tienes que volver a explicar quién es el cliente.

Plantilla Custom Instructions (copíala y adáptala)

Eres un asistente contable senior trabajando para mi firma con el cliente PYME ABC SAS (NIT 900.123.456-7), comercio textil, régimen ordinario, Grupo 2 NIIF, software World Office. REGLAS DE TRABAJO: 1. Toda cifra de pesos colombianos en formato $1.234.567 (punto como miles). 2. Toda norma citada debe llevar el artículo y la fuente: "Art. 437 ET" o "Res. 000227/2024 DIAN". 3. Si te pido clasificar cuentas del PUC, usa el Decreto 2650/1993 + adaptación NIIF Grupo 2 del cliente. 4. Si te falta un dato (NIT de un tercero, código CIIU), dilo y NO inventes. 5. Si una operación tiene tratamiento contable distinto de fiscal, explícame AMBOS y la diferencia para F2516. CONTEXTO RECURRENTE: - El cliente importa mercancía; los costos directos siguen Decreto 1165/2019. - Provisión cartera: general 1% Art. 145 ET (no individual). - Comisiones a agentes del exterior: retención 1.5% por servicios desde el exterior (Art. 408 ET). FORMATOS DE SALIDA: - Si pides papel de trabajo: tabla en Markdown con columnas Concepto / Valor / Norma. - Si pides asiento contable: bloque código con Débito | Crédito | Cuenta PUC | Tercero. - Si pides borrador de respuesta DIAN: usa formato “Señores DIAN...” en lenguaje formal.

La plantilla está pensada para un cliente PYME ordinario; para Simple, RST con varios CIIU, ESAL o personas naturales, ajustas las reglas. La estructura sigue siendo la misma.

Knowledge — qué subir y qué no

Sí subir al KnowledgeNO subir
RUT actualizado del clienteCédula del representante en imagen
Última declaración de renta firmadaClaves MUISCA, contraseñas bancos
PUC adaptado del cliente (Excel)Todos los movimientos del año (muy pesado, úsa lo del filesystem)
Memo de políticas contables NIIFFotos del local, planos, brochures comerciales
Resolución DIAN de facturación electrónicaCarpeta entera de descargas

Knowledge vs. carpeta local

El Knowledge se queda en los servidores de Anthropic (privado a tu cuenta). La carpeta local Cowork queda en tu disco. Para datos sensibles, prefiere carpeta local. Para normas, plantillas y referencias genéricas, usa Knowledge.

Lo clave de este módulo
  • Un Project por cliente. Custom Instructions + Knowledge.
  • Las Custom Instructions evitan repetir “mi cliente es...” cada chat.
  • El Knowledge va para referencias estables; el filesystem para datos vivos.

Memoria persistente y slash commands

Cowork tiene memoria automática: si en un chat le enseñas algo (“mi cliente paga las primas el 15 de junio, no el 30”), te lo recuerda en chats siguientes del mismo Project. Y los slash commands custom te dejan disparar tareas frecuentes con dos palabras.

Al finalizar este módulo
  • Entender cómo funciona la memoria automática de Cowork y cómo revisarla
  • Tener 3 slash commands listos: /cierre, /req-dian, /exogena
  • Saber editar la memoria cuando un cliente cambia de política

Memoria automática — lo que Cowork recuerda solo

Dentro de un Project, Cowork va guardando hechos que tú le enseñas: nombres, decisiones, fechas raras. La próxima vez los usa sin que se lo recuerdes. Lo puedes inspeccionar y editar en Settings → Memory del Project.

Cuándo borrar memoria

Cuando el cliente cambia de política contable (ej. pasa de Decreto 2649 a NIIF, o reclasifica una cuenta). Si no la borras, Claude va a seguir aplicando la regla vieja.

Slash commands custom — tus 3 atajos

Cowork deja crear commands propios. Vive como archivo Markdown dentro de la carpeta. Cuando escribes /cierre en el chat, Claude carga ese prompt completo y arranca.

Comando /cierre — cierre mensual de un cliente

--- name: cierre description: Cierre mensual del cliente activo --- Hazme el cierre mensual del cliente. Pasos: 1. Lee balances/{mes-actual}.xlsx 2. Compáralo contra balances/{mes-anterior}.xlsx (variación por cuenta) 3. Detecta saldos inusuales: cuentas con cambio > 30% mes-a-mes 4. Verifica que el efectivo del balance cuadra con extracto bancario (si está en facturacion-electronica\extracto-{mes}.csv) 5. Lista cuentas por cobrar a más de 90 días para evaluación de deterioro 6. Calcula provisión cartera Art. 145 ET y compárala con la que tiene 7. Genera un papel-trabajo en Markdown y guárdalo como papeles-trabajo/cierre-{mes-actual}.md

Comando /req-dian — respuesta a requerimiento

--- name: req-dian description: Borrador de respuesta a requerimiento DIAN --- Voy a pegarte un requerimiento DIAN. Tu tarea: 1. Identifica el tipo (ordinario Art. 684 / especial Art. 703 / informativo) 2. Lista LITERALMENTE qué está pidiendo (sin interpretar) 3. Para cada punto, indica qué archivo del cliente lo soporta (ruédale a balances\, declaraciones\, exogena\) 4. Redacta el borrador de respuesta en formato formal (“Señores DIAN...”) 5. Cita norma para cada argumento 6. Guarda como requerimientos-dian/respuesta-{numero-req}.docx (markdown si docx no aplica)

Comando /exogena — temporada de medios magnéticos

--- name: exogena description: Pre-armado de exógena AG del cliente --- Vas a pre-armar la exógena AG del cliente. Sigue exactamente la masterclass en escuela/exogena-claude.html. 1. Carga balances\12.xlsx (cierre de diciembre) 2. Carga las 6 declaraciones IVA del año (declaraciones\iva-300\) 3. Carga renta del año anterior si existe (declaraciones\renta-110\) 4. Genera CSV pre-llenados en exogena\: - F1001 pagos a terceros (concepto + 9 columnas) - F1005 IVA descontable / F1006 IVA generado - F1007 ingresos por tercero 5. Cuadre maestro: que F1001 + F1005 cruce con balance. NO inventes NITs. Si te falta uno, lístalo en la columna "?".
Lo clave de este módulo
  • La memoria del Project guarda hechos solo. Revísala cuando el cliente cambie de política.
  • 3 slash commands base: /cierre, /req-dian, /exogena. Crea más según tu práctica.
  • Cada comando es un Markdown corto — versíonalos en Git si quieres compartir con tu equipo.

Módulo 6 de 12

3 flujos típicos en Cowork

Los 3 momentos donde la inversión en montar todo lo anterior te paga: cierre mensual (cada mes), respuesta a requerimiento DIAN (a demanda) y temporada de exógena (febrero-abril).

Al finalizar este módulo
  • Tener un guión paso-a-paso de los 3 flujos
  • Saber cuándo intervenir y cuándo dejar correr el comando
  • Entender qué revisión humana es obligatoria

Flujo 1 — Cierre mensual

  1. Cliente te envía el balance de prueba al día 10. Lo guardas en clientes\pyme-abc\balances\2025-04.xlsx
  2. Abres el Project de PYME ABC en Cowork.
  3. Escribes /cierre. Claude lee, compara, detecta inusuales, genera papel de trabajo.
  4. Tú revisas los inusuales en 10 minutos: cuáles son reales y cuáles error de digitación.
  5. Le indicas a Claude qué ajustes proponer.
  6. Apruebas, envías ajustes al cliente, firma y archiva.

Tiempo antes: 3-4 horas por cliente. Tiempo con Cowork: 45 min para el cierre del mes con la revisión humana donde corresponde.

Flujo 2 — Requerimiento DIAN

  1. Llega el requerimiento por correo. Lo guardas como PDF en clientes\pyme-abc\requerimientos-dian\
  2. En el Project del cliente, escribes /req-dian y pegas el texto.
  3. Claude identifica el tipo, lista lo pedido, sugiere dónde está el soporte y arma borrador.
  4. Tú revisas línea por línea. Aquí NO se delega: tu firma va, tú respondes Art. 402 C.P.
  5. Ajustas, verificas que cada cifra coincide con el balance, firmas y radicas.

No deleges la respuesta final

El borrador es de Claude, la respuesta es tuya. Si firmas algo que no leíste y lleva un dato falso, el delito penal por declaración inexacta es tuyo, no de Anthropic. Cowork acelera la redacción; no reemplaza el criterio del contador firmante.

Flujo 3 — Temporada de exógena

La temporada arranca enero y cierra entre marzo y mayo según el calendario DIAN. Con la carpeta-por-cliente lista, este flujo es el que más tiempo libera.

  1. Para cada cliente: /exogena en su Project.
  2. Claude carga balance de cierre, las 6 declaraciones IVA y la renta del año anterior.
  3. Genera CSVs pre-llenados en exogena\ (F1001, F1005, F1006, F1007).
  4. Tú abres exogena.html en exogenadian.com y cargas el CSV resultante.
  5. La herramienta corre validaciones, tú revisas el log del prevalidador.
  6. Si hay errores, vuelves al chat del Project con el log y pides corrección.

Combinación con la masterclass

Este flujo es el orquestador de la masterclass Exógena con Claude. Ahí aprendes el detalle de cada formato; aquí lo aplicas sobre 10 clientes seguidos sin re-explicarle a Claude quién es cada uno.

Lo clave de este módulo
  • 3 flujos, 3 momentos. Cada uno con su slash command.
  • La revisión humana siempre va: cierre (inusuales), DIAN (línea por línea), exógena (log del prevalidador).
  • El ahorro real aparece en el segundo mes — cuando ya no tienes que armar la estructura.

Qué es una Skill y su anatomía

Una Skill es una receta que le enseñas a Claude una vez y la usas siempre —en Cowork y también en Claude Code—. Para un contador, las Skills convierten tus mejores prompts (cuadre IVA, clasificación F1001) en herramientas que se activan solas cuando la tarea lo amerita.

Al finalizar este módulo
  • Distinguir Skill vs. prompt suelto vs. slash command
  • Conocer la estructura de carpeta de una skill (SKILL.md + references + examples)
  • Saber escribir el description que hace que Claude la active en el momento correcto

Skill vs. prompt vs. slash command

 Prompt sueltoSlash commandSkill
Vive enTu memoria / nota sueltaUn archivo .md en la carpetaUna carpeta dedicada con SKILL.md + scripts + ejemplos
ReusableNo (lo reescribes cada vez)Sí con un comandoSí y composable con otras skills
Sirve paraTareas únicasTareas frecuentes en un ProjectTareas que se repiten en TODOS los clientes y quieres compartir con tu equipo
Tamaño1-5 líneas10-30 líneas50-500 líneas + ejemplos + tests

Criterio para subir de slash a skill

Si te vas a copiar el slash command de un cliente a otro y le tienes que ajustar variables cada vez, ya merece ser skill. Las skills se parametrizan y se cargan donde sea (no viven dentro de un Project específico) — y la misma carpeta sirve en Cowork y en Claude Code.

3 casos contables donde una Skill paga rapidísimo

Clasificador F1001

El archivo SKILL.md — tres bloques

SKILL.md tiene tres bloques bien definidos: frontmatter (nombre + descripción para que Claude sepa cuándo activarla), instrucciones (qué hacer paso a paso) y contratos (qué espera de entrada y qué entrega de salida).

--- name: clasificador-exogena-co description: Clasifica cuentas del PUC colombiano a conceptos del F1001   de información exógena DIAN AG. Usar cuando el usuario quiera   pre-armar el F1001 desde un balance o un auxiliar. --- # Clasificador Exógena CO ## Cuándo activarte Cuando el usuario pegue un auxiliar contable, un balance o un listado de cuentas y mencione: F1001, exógena, medios magnéticos, pagos a terceros o Art. 631 ET. ## Lo que haces 1. Lee la entrada (Excel/CSV con cuentas y movimientos). 2. Para cada cuenta del grupo 5 (gastos), 7 (costos) y 13xxxx (deudores no operacionales relevantes),    asigna uno de los 28 conceptos del F1001 (ver references/conceptos-f1001.md). 3. Para conceptos con tarifas IVA o retención diferenciadas, desglosa en columnas    (ver el detalle de las 9 columnas en references/conceptos-f1001.md). 4. Si una cuenta no mapea a ningún concepto, muévela a "Concepto 5066 — Otros pagos". 5. Si una cuenta DEBE excluirse del F1001 (ej. nómina ya en F2276), muévela a "exclusión" con motivo. ## Lo que entregas Una tabla Markdown con columnas: | Código PUC | Nombre cuenta | Concepto F1001 | Justificación (norma) | Valor base | Valor IVA mayor | Valor reten | ## Restricciones - NUNCA inventes el NIT de un tercero. Si falta, marca "?" y lístalo aparte. - SIEMPRE cita el artículo del ET o de la Res. 000227 que justifica la clasificación. - Si una cuenta tiene movimiento por debajo del tope reportable, márcala "< tope: no reportable". - Para personas naturales asalariadas (concepto 5004), redondear a múltiplos de mil según especificación MUISCA.

Por qué la description importa tanto

Claude lee el description del frontmatter de TODAS tus skills al inicio del chat y decide cuáles activar. Si tu description dice “clasifica cosas”, no se va a activar nunca. Si dice “Clasifica cuentas del PUC colombiano a conceptos del F1001... Usar cuando...” Claude sabe exactamente cuándo entrar.

Errores comunes en la description

1) Demasiado genérica (“Ayuda con exógena”). 2) En inglés en una skill española. 3) Sin la frase “Usar cuando...” (Claude no infiere el trigger sin ella). 4) Más larga que 2 frases (se vuelve ruido).

References — donde vive la norma

La carpeta references/ es para documentos largos que NO quieres cargar siempre, sino solo cuando la skill los necesita. La Res. 000227/2024 completa son 80 páginas; resumida en res-227-2024.md son 6. Cargas la versión resumida y Claude trabaja sobre eso.

Cuándo actualizar references

Cada vez que la DIAN saca una resolución modificatoria. Para AG actual, las modificatorias 000233 y 000237 de 2025 ya están integradas; cuando salga la próxima toca actualizar.

Lo clave de este módulo
  • Skill = receta + scripts + ejemplos. Vive aparte del Project y sirve en Cowork y Claude Code.
  • Se justifica cuando la lógica se repite en todos tus clientes.
  • El description es el trigger: claro, específico, en español, con “Usar cuando...”.
  • Las references se actualizan cuando cambia la norma.

Skill #2

cuadre-iva-co

Lee las 6 declaraciones IVA 300 bimestrales (o cuatrimestrales/anual según el caso) y las cuadra contra balance: 2408 vs IVA generado, 1355 vs IVA descontable, 2367 vs retenciones de IVA practicadas.

Art. 437 ET Art. 488 ET Decreto 1625/2016

SKILL.md del cuadre IVA

--- name: cuadre-iva-co description: Cuadra las declaraciones IVA 300 del año contra el balance contable   colombiano. Activar cuando el usuario quiera revisar consistencia de IVA generado,   descontable, retenciones y saldos antes de cierre o exógena. --- # Cuadre IVA CO ## Pasos 1. Lee las declaraciones IVA: en orden, F300 bimestre 1 a 6 (o equivalente anual RST). 2. Lee el balance acumulado de diciembre. 3. Por cada bimestre, calcula:    a) IVA generado del 300 (renglones 27 al 58) vs movimiento de la cuenta 2408 en el bimestre.    b) IVA descontable del 300 (renglones 67 al 79) vs movimiento de la cuenta 1355 en el bimestre.    c) Retenciones de IVA practicadas (renglones 91-92) vs movimiento de 2367.    d) Saldo a favor o pagar (renglones 93-96) vs cuenta 1355.95 o 2408 saldo final. 4. Reporta diferencias > 1% en cada celda con causa probable (digitación / norma / temporal). 5. Si hay saldos a favor: verifica que se imputaron al siguiente bimestre o se solicitaron. ## Contrato de salida Tabla por bimestre: | Bimestre | Concepto | Declaración 300 | Balance | Diferencia | Causa probable | Acción sugerida | ## Restricciones - Si el cliente es RST: la declaración es anual (F260), no bimestral. Usar references/cuadre-rst.md. - Si maneja AIU: la base de IVA cambia — ver references/aiu.md. - NO sugerir "presentar corrección" sin revisar firmeza (Art. 714 ET).

Saldos a favor y firmeza

La skill puede detectar un descuadre de hace 3 bimestres, pero presentar corrección con sanción del Art. 644 ET es decisión tuya. La firmeza juega: una declaración con saldo a favor solicitado en devolución queda en firme distinto que una con saldo a pagar. La skill no debe sugerir corrección automática.

Detecta el tipo de error (categoriza con references/catalogo-errores.md):    - Estructurales (tag mal formado, encoding)    - De negocio (NIT inválido, concepto fuera de catálogo)    - De cruce (suma de subconceptos no coincide con valor total)    - De tope (valor por debajo del mínimo reportable) 2. ## E001 — NIT del tercero no válido (dígito verificación) Causa: NIT mal digitado o dígito de verificación incorrecto. Acción: validar con consulta NIT DIAN. Si NIT correcto, ajustar DV. ## E002 — Suma de subconceptos ≠ valor reportado Causa: típico cuando se llena valor base sin desglosar IVA mayor / reten. Acción: revisar columnas 4-9 del F1001, verificar que sumen al total. ## E003 — Concepto fuera del catálogo del año Causa: código de concepto que cambió entre AG (DIAN actualiza cada año). Acción: validar concepto vigente según Res. del año (ej. AG 2025 = Res. 000227/2024). ## E004 — TipoDoc inconsistente con NIT Causa: NIT formato persona jurídica con tipoDoc=CC. Típico en extranjeros (CE/PEP/PPT). Acción: revisar tipoDoc; si es extranjero, usar 21=CE / 41=PAS / 47=PEP / 48=PPT. ## E005 — Dirección del tercero excede longitud Causa: direcciones largas con caracteres especiales. Acción: simplificar dirección a máx 100 caracteres sin acentos ni &. ## E006 — Concepto requerido según responsabilidades RUT Causa: cliente tiene resp 05 pero no reportó F1005 (IVA descontable). Acción: revisar responsabilidades RUT vs formatos enviados. ## E007 — Suma F1001 vs F1011 difiere Causa: pagos por conceptos sumados ≠ total renta de ese tipo. Acción: verificar cruces F1001 / F1004 / F1011 con renta del cliente. ## E008 — País incorrecto para pagos al exterior Causa: código de país no válido en catálogo DIAN. Acción: usar catálogo ISO 3 letras según especificación. ## E009 — Valor negativo en concepto que no admite negativo Causa: devolución o NC mal aplicada al concepto positivo. Acción: reportar como reducción del bimestre anterior o reclasificar concepto. ## E010 — Fecha fuera del período declarado Causa: movimiento de enero del año siguiente colado en el AG actual. Claude Code para contadores — cuándo pasar a la terminal

Claude Code es el agente de Anthropic para trabajo técnico: en vez de conversar sobre un archivo, ejecuta — corre comandos, transforma 40 Excel en lote, escribe scripts y los prueba. No necesitas ser programador para sacarle provecho; necesitas saber cuándo usarlo en vez de Cowork.

Al finalizar este módulo
  • Saber cuándo una tarea es de Cowork y cuándo es de Claude Code
  • Tenerlo instalado y haber corrido tu primera tarea en lote
  • Entender el modelo de permisos (Claude propone, tú apruebas)

Cowork vs. Claude Code — la regla práctica

La tarea es...Úsalo enPor qué
Analizar UN balance, redactar UNA respuesta, revisar UN cierreCoworkEs conversación sobre archivos: lees, preguntas, ajustas.
Hacer LO MISMO sobre 40 archivos (renombrar PDFs de clientes, consolidar 12 balances mensuales en uno, convertir todos los extractos a CSV)Claude CodeEs ejecución en lote: Claude escribe el script, lo corre, verifica el resultado.
Construir una herramienta que vas a usar todos los meses (liquidador, conciliador, generador de certificados)Claude CodeEs desarrollo de software — y la profundización va en Desarrollo para Contadores.

El test de la repetición

Si al describir la tarea dices “para cada archivo...” o “todos los meses hago...”, es Claude Code. Si dices “mira este archivo y dime...”, es Cowork.

Dónde corre y cómo se instala

Claude Code existe en varios formatos: CLI en la terminal (el original), app de escritorio (Windows/Mac), web en claude.ai/code y extensiones de IDE (VS Code, JetBrains). Para un contador sin terminal, la app de escritorio o la web son la puerta de entrada más amable; usa la misma cuenta Pro de Claude que ya tienes.

Una vez dentro, el flujo es el mismo de Cowork pero orientado a acción: le señalas una carpeta de trabajo, le describes la tarea, y Claude propone los pasos y los ejecuta.

Tu primera tarea en lote

Caso real de temporada: el cliente te mandó 12 balances mensuales en 12 Excel con formatos ligeramente distintos, y necesitas un consolidado anual. En Claude Code, parado sobre la carpeta:

En esta carpeta hay 12 archivos balance-2025-01.xlsx a balance-2025-12.xlsx. Cada uno tiene columnas Código PUC / Nombre / Débito / Crédito / Saldo, aunque algunos meses traen la cabecera en la fila 3 y otros en la 5. 1. Lee los 12 y normalízalos a la misma estructura. 2. Genera consolidado-2025.xlsx con una hoja por mes + una hoja "Anual" con el acumulado por cuenta a nivel 4 dígitos. 3. Avísame si algún mes tiene cuentas que no aparecen en los demás. NO modifiques los archivos originales.

Claude escribe el script (Python con openpyxl o similar), te muestra qué va a hacer, lo corre y te entrega el consolidado. Lo que a mano toma una tarde, sale en minutos — y el script queda guardado para el año siguiente.

El modelo de permisos: Claude propone, tú apruebas

A diferencia del chat, Claude Code puede ejecutar comandos y modificar archivos. Por eso trabaja con permisos: antes de correr un comando o editar un archivo te lo muestra y espera tu aprobación. Reglas de oro para un contador:

  • Trabaja sobre copias. Antes de una tarea en lote, duplica la carpeta. Los originales del cliente no se tocan.
  • Lee lo que apruebas. Si propone borrar o sobreescribir algo y no entiendes por qué, pregunta antes de aprobar.
  • No actives el modo automático (aprobar todo sin preguntar) con datos de clientes hasta que lleves semanas usándolo.

Error común

Darle acceso a la carpeta raíz con TODOS los clientes para una tarea de uno solo. Igual que en Cowork (Módulo 2): carpeta dedicada por tarea, y Claude solo ve lo que necesita.

Tus skills también funcionan aquí

Las skills que montaste en los Módulos 7-9 viven en ~/.claude/skills/ — y Claude Code las lee del mismo lugar. El clasificador F1001 que usas conversando en Cowork es el mismo que Claude Code puede invocar dentro de un script de lote para procesar los 15 clientes de una sentada (con tu revisión entre cliente y cliente, como en el pipeline del Módulo 10).

Si esto te abre el apetito de construir herramientas propias —tu liquidador, tu conciliador, tu página— el camino completo (Git, Next.js, TypeScript, cómo pedirle bien las cosas a Claude Code) está en Desarrollo para Contadores.

Lo clave de este módulo
  • Regla: conversación sobre UN archivo = Cowork; ejecución sobre MUCHOS = Claude Code.
  • Corre en terminal, escritorio, web e IDE — misma cuenta Pro.
  • Permisos: propone antes de ejecutar. Copias primero, originales intactos.
  • Las mismas skills de ~/.claude/skills/ sirven en Cowork y en Code.

Backup de la carpeta Claude-Trabajo

La carpeta local es valiosa: tiene la configuración de tu firma (slash commands, plantillas, fichas _cliente.md) y los papeles de trabajo. Si se quema el disco, pierdes meses.

Opción A — Git privado

Repositorio privado en GitHub o GitLab. Solo lo de configuración (slash commands, plantillas). Los balances reales no van a Git (gitignore balances/, declaraciones/, requerimientos-dian/).

Opción B — Drive privado

OneDrive personal o Google Drive personal cifrado. Sirve para los datos reales, pero nunca un Drive corporativo del cliente.

Opción C — Disco externo

El más simple. SSD externo cifrado, backup semanal manual. Mantienes todo offline.
