Qué es una Skill y cuándo construir una

Una Skill es una receta que le enseñas a Claude una vez y la usas siempre. Para un contador, las Skills convierten tus mejores prompts (cuadre IVA, clasificación F1001) en herramientas que disparas con un comando.

Al finalizar este módulo
  • Distinguir Skill vs. prompt suelto vs. slash command
  • Saber el criterio para decidir esto merece ser skill
  • Tener listo el caso que vas a convertir en skill en el Módulo 3
Promesa del curso

Vas a salir con 3 Skills funcionando para tu práctica contable colombiana: clasificador de exógena, cuadre IVA y validador del prevalidador.

Las 3 son cierre real: no ejemplos genéricos. La clasificador-exogena-co mapea PUC a concepto F1001 con la Res. 000227/2024. La cuadre-iva-co cuadra IVA 300 contra balance. La validador-prevalidador lee logs del prevalidador V9 y los traduce a contador.

Criterio para subir de slash a skill

Si te vas a copiar el slash command de un cliente a otro y le tienes que ajustar variables cada vez, ya merece ser skill. Las skills se parametrizan y se cargan donde sea (no viven dentro de un Project específico).

3 casos contables donde una Skill paga rapidísimo

Clasificador F1001

Mapear cuentas del PUC a uno de los 28 conceptos del F1001. Lo haces igual para los 15 clientes. Skill obligatoria.

Cuadre IVA vs. balance

IVA generado del 300 contra cuenta 2408. IVA descontable contra cuenta 1355. Misma lógica, distintas cifras. Skill obligatoria.

Lectura del prevalidador

El log del prevalidador es feo y enumera errores técnicos. Una skill que los traduce a “te falta el NIT del tercero X en F1001” te ahorra 30 min por iteración. Skill obligatoria.

Lo clave de este módulo
  • Skill = receta + scripts + ejemplos. Vive aparte del Project.
  • Se justifica cuando la lógica se repite en todos tus clientes.
  • 3 candidatos obvios: clasificar F1001, cuadrar IVA, leer prevalidador.

Anatomía de una Skill

Una skill es una carpeta con un archivo principal (SKILL.md), scripts opcionales y ejemplos. Esta estructura no es opcional: Claude busca exactamente estos archivos para entender qué hace la skill y cuándo activarla.

Al finalizar este módulo
  • Tener la estructura de carpetas para tu primera skill
  • Saber qué va en SKILL.md (frontmatter + descripción + instrucciones)
  • Entender la diferencia entre instrucción y script ejecutable

Estructura de la carpeta

skills\clasificador-exogena-co\   SKILL.md # El archivo principal: descripción + instrucciones   references\     res-227-2024.md # Norma DIAN resumida     conceptos-f1001.md # Los 28 conceptos del F1001   examples\     caso-pyme.md # Ejemplo entrada/salida real     caso-esal.md   scripts\ # Opcional: scripts ejecutables Python/JS     verificar_nit.py

El archivo SKILL.md — tres bloques

SKILL.md tiene tres bloques bien definidos: frontmatter (nombre + descripción para que Claude sepa cuándo activarla), instrucciones (qué hacer paso a paso) y contratos (qué espera de entrada y qué entrega de salida).

--- name: clasificador-exogena-co description: Clasifica cuentas del PUC colombiano a conceptos del F1001   de información exógena DIAN AG. Usar cuando el usuario quiera   pre-armar el F1001 desde un balance o un auxiliar. --- # Clasificador Exógena CO ## Cuándo activarte Cuando el usuario pegue un auxiliar contable, un balance o un listado de cuentas y mencione: F1001, exógena, medios magnéticos, pagos a terceros o Art. 631 ET. ## Lo que haces 1. Lee la entrada (Excel/CSV con cuentas y movimientos). 2. Para cada cuenta del grupo 5 (gastos), 7 (costos) y 13xxxx (deudores no operacionales relevantes),    asigna uno de los 28 conceptos del F1001 (ver references/conceptos-f1001.md). 3. Para conceptos con tarifas IVA o retención diferenciadas, desglosa en columnas    (ver el detalle de las 9 columnas en references/conceptos-f1001.md). 4. Si una cuenta no mapea a ningún concepto, muévela a "Concepto 5066 — Otros pagos". 5. Si una cuenta DEBE excluirse del F1001 (ej. nómina ya en F2276), muévela a "exclusión" con motivo. ## Lo que entregas Una tabla Markdown con columnas: | Código PUC | Nombre cuenta | Concepto F1001 | Justificación (norma) | Valor base | Valor IVA mayor | Valor reten | ## Restricciones - NUNCA inventes el NIT de un tercero. Si falta, marca "?" y lístalo aparte. - SIEMPRE cita el artículo del ET o de la Res. 000227 que justifica la clasificación. - Si una cuenta tiene movimiento por debajo del tope reportable, márcala "< tope: no reportable". - Para personas naturales asalariadas (concepto 5004), redondear a múltiplos de mil según especificación MUISCA.

Por qué la description importa tanto

Claude lee el description del frontmatter de TODAS tus skills al inicio del chat y decide cuáles activar. Si tu description dice “clasifica cosas”, no se va a activar nunca. Si dice “Clasifica cuentas del PUC colombiano a conceptos del F1001... Usar cuando...” Claude sabe exactamente cuándo entrar.

Errores comunes en la description

1) Demasiado genérica (“Ayuda con exógena”). 2) En inglés en una skill española. 3) Sin la frase “Usar cuando...” (Claude no infiere el trigger sin ella). 4) Más larga que 2 frases (se vuelve ruido).

References — donde vive la norma

La carpeta references/ es para documentos largos que NO quieres cargar siempre, sino solo cuando la skill los necesita. La Res. 000227/2024 completa son 80 páginas; resumida en res-227-2024.md son 6. Cargas la versión resumida y Claude trabaja sobre eso.

Cuándo actualizar references

Cada vez que la DIAN saca una resolución modificatoria. Para AG actual, las modificatorias 000233 y 000237 de 2025 ya están integradas; cuando salga la próxima en 2027 toca actualizar.

Lo clave de este módulo
  • Una skill = carpeta + SKILL.md + opcionalmente references/, examples/, scripts/.
  • El description es el trigger: claro, específico, en español, con “Usar cuando...”.
  • Las references se actualizan cuando cambia la norma.

Tu primera Skill: clasificador-exogena-co

Construimos la primera skill completa de principio a fin. Es la que más tiempo libera durante temporada (enero-abril): clasifica automáticamente cuentas del PUC a los 28 conceptos del F1001 con la norma DIAN vigente.

Skill #1

clasificador-exogena-co

Lee un auxiliar contable y devuelve una tabla con concepto F1001 asignado por cuenta, valor base, valor IVA mayor, valor retención y norma que justifica la clasificación.

Res. 000227/2024 Art. 631 ET Decreto 2650/1993 (PUC)

Paso 1 — SKILL.md (copíala y adáptala)

Ya la vimos en el Módulo 2. Solo asegúrate de pegarla en skills\clasificador-exogena-co\SKILL.md.

Paso 2 — references/conceptos-f1001.md (los 28 conceptos)

# Conceptos del F1001 — AG vigente ## Fuente: Res. 000227/2024 DIAN mod. por 000233 y 000237/2025 ## Conceptos más usados (ordenados por frecuencia en PYMES CO) 5001 — Salarios, prestaciones sociales y demás pagos laborales.    Cuentas PUC típicas: 5105 / 5205 / 7105.    NO reportar si está en F2276 Nómina (excepto pagos no reportados ahí). 5004 — Honorarios profesionales personas naturales / jurídicas.    Cuentas PUC: 5110, 5210, 7110.    Desglose: base de retención 10% PJ / 11% PN no declarante / 4% RST. 5006 — Servicios en general (no honorarios).    Cuentas PUC: 5135 (mantenimiento), 5140 (gastos legales), 5145.    Reten estandar 4-6% según declarante. 5008 — Arrendamientos.    Cuenta PUC: 5120.    Reten 3.5% (bienes inmuebles), 4% (bienes muebles). 5010 — Compra de activos fijos.    Cuentas PUC: 152x, 154x, 156x.    Concepto distinto si es por leasing financiero (1465 / 2335). 5011 — Compra de inventarios (mercancías).    Cuentas PUC: 1435 / 6135 / 7405.    Si es importación: incluir aranceles e IVA importación en valor base. 5013 — Compra de bienes no inventario (suministros, papelería).    Cuentas PUC: 5195, 7195. 5014 — Pagos a comisionistas, intermediarios.    Cuenta PUC: 5125.    Reten 10% (PJ y PN declarante). 5015 — Aportes parafiscales (SENA, ICBF, CCF).    Cuenta PUC: 2370. 5018 — Aportes seguridad social (salud, pensión, ARL).    Cuentas PUC: 2380. 5020 — Intereses y rendimientos financieros pagados.    Cuentas PUC: 5305, 7305. 5022 — Donaciones.    Cuentas PUC: 5295.    Tener cuidado: descuento renta Art. 257 ET requiere ESAL beneficiaria. 5024 — Impuestos efectivamente pagados (predial, ICA, vehic).    Cuenta PUC: 5115. 5027 — Servicios públicos.    Cuenta PUC: 5135. 5029 — Pagos al exterior (no servicios técnicos).    Reten 33%-35% según concepto y CDI. 5066 — Otros pagos — CAJÓN FINAL. Usar SOLO si ningún otro encaja.

Por qué solo los más usados

El F1001 vigente tiene 28 conceptos. En un PYME ordinario CO usas activamente unos 15. Esta references documenta los 15 críticos para que la skill clasifique correcto; los exotícos (operaciones con el exterior por TFC, pagos a entidades estatales) los agregas según te aparezcan.

Paso 4 — Probar la skill

En Cowork, abre cualquier Project. Si tienes la skill en ~/.claude/skills/clasificador-exogena-co/ (o en la carpeta de skills del Project), basta con escribir:

Te paso el auxiliar de mi cliente XYZ del 2025. Pre-arma el F1001. [pegas o adjuntas la tabla]

Claude detecta que mencionas F1001, lee la description de tu skill, la activa, carga las references y produce la tabla según el contrato del SKILL.md. Tú revisas las celdas marcadas con “?” (NITs faltantes) y los casos donde la skill usó el concepto-cajón 5066 (suelen necesitar criterio humano).

Lo clave de este módulo
  • 3 archivos: SKILL.md, references/conceptos-f1001.md, examples/caso-pyme.md.
  • El ejemplo entrada-salida es lo que más sube la calidad.
  • NITs faltantes y concepto 5066 son las celdas donde tú intervienes.

Skill 2: cuadre-iva-co

Cuadra las 6 declaraciones IVA 300 del año contra el balance: IVA generado vs cuenta 2408, IVA descontable vs cuenta 1355. Te marca los meses descuadrados y por cuánto.

Al finalizar este módulo
  • Tener skills\cuadre-iva-co\ funcionando con tu cliente
  • Entender el modelo de saldos a favor vs por pagar y cómo afecta el cuadre
  • Saber distinguir error de digitación vs error normativo en la diferencia
Skill #2

cuadre-iva-co

Lee las 6 declaraciones IVA 300 bimestrales (o cuatrimestrales/anual según RST) y las cuadra contra balance: 2408 vs IVA generado, 1355 vs IVA descontable, 1355.95 vs retenciones de IVA.

Art. 437 ET Art. 488 ET Decreto 1625/2016

SKILL.md

--- name: cuadre-iva-co description: Cuadra las declaraciones IVA 300 del año contra el balance contable   colombiano. Activar cuando el usuario quiera revisar consistencia de IVA generado,   descontable, retenciones y saldos antes de cierre o exógena. --- # Cuadre IVA CO ## Pasos 1. Lee las declaraciones IVA: en orden, F300 bimestre 1 a 6 (o equivalente anual RST). 2. Lee el balance acumulado de diciembre. 3. Por cada bimestre, calcula:    a) IVA generado del 300 (renglones 27 al 58) vs movimiento de la cuenta 2408 en el bimestre.    b) IVA descontable del 300 (renglones 67 al 79) vs movimiento de la cuenta 1355 en el bimestre.    c) Retenciones de IVA practicadas (renglones 91-92) vs movimiento de 2367.    d) Saldo a favor o pagar (renglones 93-96) vs cuenta 1355.95 o 2408 saldo final. 4. Reporta diferencias > 1% en cada celda con causa probable (digitación / norma / temporal). 5. Si hay saldos a favor: verifica que se imputaron al siguiente bimestre o se solicitaron. ## Contrato de salida Tabla por bimestre: | Bimestre | Concepto | Declaración 300 | Balance | Diferencia | Causa probable | Acción sugerida | ## Restricciones - Si el cliente es RST: la declaración es anual (F260), no bimestral. Usar references/cuadre-rst.md. - Si maneja AIU: la base de IVA cambia — ver references/aiu.md. - NO sugerir "presentar corrección" sin revisar firmeza (Art. 714 ET).

Saldos a favor y firmeza

La skill puede detectar un descuadre de hace 3 bimestres, pero presentar corrección con sanción del Art. 644 ET es decisión tuya. La firmeza juega: una declaración con saldo a favor solicitado en devolución queda en firme distinto que una con saldo a pagar. La skill no debe sugerir corrección automática.

Lo clave de este módulo
  • 4 cruces por bimestre: generado, descontable, retenido, saldo.
  • Diferencias > 1% se reportan con causa probable.
  • La acción correctiva la decide el contador, no la skill.

Skill 3: validador-prevalidador

El prevalidador V9 escupe logs técnicos en formato XML/texto crudo. Esta skill los traduce a errores en lenguaje de contador con la acción correctiva exacta.

Al finalizar este módulo
  • Tener skills\validador-prevalidador\ con catalogo de errores comunes
  • Ahorrar al menos 30 min por iteración del prevalidador
  • Saber cuáles errores reincidentes ameritan ajustar tu plantilla base
Skill #3

validador-prevalidador

Lee el log del prevalidador V9 (XML o log de texto) y devuelve cada error en formato “Renglon X del F1001: falta NIT del tercero Y; solución: pasar a registro #N el código CIIU”.

Prevalidador DIAN V9 Res. 000227/2024

SKILL.md

--- name: validador-prevalidador description: Traduce errores del prevalidador V9 a lenguaje contable con la acción   correctiva exacta. Activar cuando el usuario pegue un log de error o XML del prevalidador. --- # Validador del Prevalidador V9 ## Cuándo activarte Cuando el usuario pegue un texto que contenga: “PrevalidadorV9”, “EXCG”, “error de validación”, o un bloque XML con tags <Errores> o <ValidationResult>. ## Pasos 1. Detecta el tipo de error (categoriza con references/catalogo-errores.md):    - Estructurales (tag mal formado, encoding)    - De negocio (NIT inválido, concepto fuera de catálogo)    - De cruce (suma de subconceptos no coincide con valor total)    - De tope (valor por debajo del mínimo reportable) 2. Para cada error, identifica:    - Formato afectado (F1001/F1005/...)    - Registro / renglon    - Causa probable    - Acción correctiva paso a paso 3. Prioriza: bloquean transmisión vs son advertencias. ## Contrato de salida | # | Formato | Registro | Tipo error | Mensaje crudo | Traducción contable | Acción | |---|------|------|------|------|------|------| ## Restricciones - NUNCA sugieras “ignorar el error” sin verificar que es advertencia, no bloqueante. - Si el error es de NIT que no existe en RUES, sugerir consulta NIT antes de inventar tipoDoc. - Para errores recurrentes en el mismo cliente, sugerir actualizar plantilla maestra del cliente.

references/catalogo-errores.md (los 10 más comunes)

## E001 — NIT del tercero no válido (dígito verificación) Causa: NIT mal digitado o dígito de verificación incorrecto. Acción: validar con consulta NIT DIAN. Si NIT correcto, ajustar DV. ## E002 — Suma de subconceptos ≠ valor reportado Causa: típico cuando se llena valor base sin desglosar IVA mayor / reten. Acción: revisar columnas 4-9 del F1001, verificar que sumen al total. ## E003 — Concepto fuera del catálogo del año Causa: código de concepto que cambió entre AG (DIAN actualiza cada año). Acción: validar concepto vigente según Res. del año (ej. AG 2025 = Res. 000227/2024). ## E004 — TipoDoc inconsistente con NIT Causa: NIT formato persona jurídica con tipoDoc=CC. Típico en extranjeros (CE/PEP/PPT). Acción: revisar tipoDoc; si es extranjero, usar 21=CE / 41=PAS / 47=PEP / 48=PPT. ## E005 — Dirección del tercero excede longitud Causa: direcciones largas con caracteres especiales. Prevalidador trunca o rechaza. Acción: simplificar dirección a máx 100 caracteres sin acentos ni &. ## E006 — Concepto requerido según responsabilidades RUT Causa: cliente tiene resp 05 pero no reportó F1005 (IVA descontable). Acción: revisar responsabilidades RUT vs formatos enviados. ## E007 — Suma F1001 vs F1011 difiere Causa: pagos por conceptos sumados ≠ total renta de ese tipo. Acción: verificar cruces F1001 / F1004 / F1011 con renta del cliente. ## E008 — País incorrecto para pagos al exterior Causa: código de país no válido en catálogo DIAN. Acción: usar catálogo ISO 3 letras según especificación. ## E009 — Valor negativo en concepto que no admite negativo Causa: devolución o NC mal aplicada al concepto positivo. Acción: reportar como reducción del bimestre anterior o reclasificar concepto. ## E010 — Fecha fuera del período declarado Causa: movimiento de enero del año siguiente colado en el AG actual. Acción: filtrar fechas del balance al período exacto antes de generar XML.
Lo clave de este módulo
  • El catálogo de 10 errores cubre el 90% de los casos en CO.
  • La salida tabulada hace que sepas cuáles bloquean y cuáles son advertencias.
  • Errores recurrentes en un cliente = señal para ajustar plantilla maestra.

Encadenar skills — el pipeline exógena

Una skill sola te ahorra una tarea. Tres skills encadenadas te ahorran un proceso. Encadenamos las 3 anteriores en un pipeline: balance → clasificador → cuadre IVA → XML → validador.

Al finalizar este módulo
  • Entender cómo Claude activa varias skills en una sola conversación
  • Tener el pipeline corriendo end-to-end en uno de tus clientes
  • Saber dónde necesita supervisión humana entre paso y paso

Cada flecha es un punto de revisión humana. La idea NO es darle “haz exo de XYZ” y que termine sin ver. Es que cada skill haga su parte, tú revises, y pases a la siguiente.

Prompt orquestador

Tu Project del cliente puede tener un slash command /exogena-full que orquesta las 3 skills en orden. Lo construyes apilando los disparadores naturales:

--- name: exogena-full description: Pipeline completo de exógena para el cliente activo --- Vas a procesar la exógena AG completa del cliente. Tres fases, una revisión humana entre cada una. FASE 1 — Clasificación F1001 Carga balances/12.xlsx y usa la skill clasificador-exogena-co. Entrega tabla y PÁRATE. Espera mi OK antes de pasar a la Fase 2. FASE 2 — Cuadre IVA Carga las declaraciones declaraciones/iva-300/*.pdf y usa la skill cuadre-iva-co. Cruza el resultado con lo de la Fase 1 (IVA descontable de F1005 debe casar con cuenta 1355). Entrega tabla de cuadre y PÁRATE. FASE 3 — Validación previa Genera los CSVs/XMLs para el prevalidador. Cuando yo te pase el log del prevalidador, usa la skill validador-prevalidador. NO ejecutes las fases sin mi confirmación entre cada una.

PÁRATE entre fases — importante

La instrucción “PÁRATE” es lo que mantiene a Claude en modo asistente. Sin eso, intentará hacer todo en un solo bloque y te llega un papel de trabajo de 12 páginas que ya nadie revisa.

Lo clave de este módulo
  • Pipeline = orquestador que llama skills en orden con paradas humanas.
  • “PÁRATE” entre fases evita el chorro de 12 páginas sin revisar.
  • Cada cliente puede tener su propio orquestador con ajustes (RST, ESAL, etc).

Skills para tu firma — versionar y compartir

Si trabajas solo, basta con tener las skills en tu disco. Si tienes equipo o aspiras a tener, las skills van en repositorio Git compartido. Así cuando la DIAN cambia la Resolución y tú actualizas la references, todo tu equipo se beneficia.

Al finalizar este módulo
  • Tener tus 3 skills en un repo Git privado (GitHub/GitLab)
  • Saber cómo cargar las skills compartidas en el Cowork de un nuevo miembro del equipo
  • Tener un proceso ligero de PR/revisión cuando alguien propone cambios

Estructura del repo

firma-claude-skills\ # Repo Git privado   README.md # Cómo instalar, cómo contribuir   CHANGELOG.md # Cada cambio normativo aplicado   skills\     clasificador-exogena-co\     cuadre-iva-co\     validador-prevalidador\   .github\     PULL_REQUEST_TEMPLATE.md # Plantilla para cambios

Onboarding de un nuevo miembro del equipo

  1. Le das acceso al repo privado.
  2. Clona en su carpeta de Claude: git clone <url> ~/.claude/skills-firma
  3. Configura los nombres de las skills en Cowork (Settings → Skills Path).
  4. Listo. Las 3 skills aparecen cuando trabaje cualquier cliente.

Política de cambios

Cambio en una skill = PR al repo. Quien revisa: el contador senior. Revisión mínima: que la description sigue siendo clara, que el SKILL.md cita la norma actualizada y que el CHANGELOG.md menciona la resolución que justificó el cambio.

Cuando cambia la normativa

Tipo de cambio normativoAcción en las skills
Res. modificatoria de exógena (anual)Actualizar references/ de clasificador-exogena-co y validador-prevalidador
Nueva tarifa UVT (anual)Si las skills tienen valores en UVT, dejar variable. Si tienen pesos, actualizar.
Reforma tributaria mayor (Ley)Revisión manual completa. No todas las skills pueden seguir siendo válidas.
Decreto reglamentario nuevoAgregar al references correspondiente.
Lo clave de este módulo
  • Skills en Git privado para versión + propagación al equipo.
  • PR template + CHANGELOG mantienen trazabilidad normativa.
  • Cambios anuales en references; cambios mayores requieren revisión total.
